# AGENTS.md

Правила для ИИ-агента, который меняет проект `voice-sales-log`.

## Перед началом работы

Перед изменением кода обязательно прочитать:

- `README.md`
- `docs/specs.md`
- `docs/rules.md`
- `docs/workplan.md`

## Главная граница проекта

- Не добавлять лишние функции без необходимости.
- Не превращать проект в CRM, склад, каталог товаров, кассовую систему или систему онлайн-оплаты.
- Сохранять простую идею проекта: голосовое сообщение -> текст -> структурированные позиции продажи -> запись -> отчёт.
- Отчёты должны оставаться простыми: товар, количество, выручка, итоговая сумма.
- ИИ не должен выдумывать товары, цены, категории, аналитику или остатки.

## Документация

- После каждого изменения обновлять `CHANGELOG.md`.
- Если изменилась логика проекта, обновлять `docs/specs.md`.
- Если изменилась архитектура, обновлять `docs/architecture.md`.
- Если выполнена задача, отмечать её в `docs/workplan.md`.
- Если добавлена новая фича, описать её в `docs/features.md` и при необходимости в `docs/specs.md`.
- Не менять структуру папок без обновления документации.

## Безопасность и env

- Все переменные окружения добавлять в `.env.example`.
- Локальные значения хранить в `.env.local`; этот файл не коммитить.
- Не хранить секретные ключи в коде.
- Не использовать hardcode токенов.
- Не использовать `SUPABASE_SERVICE_ROLE_KEY` в браузерном коде.

## Код и запуск

- Не ломать существующие команды запуска.
- Не удалять рабочий код без причины.
- Все записи должны иметь `created_at`.
- Все голосовые записи и продажи должны быть связаны с продавцом, если продавец известен.
- Все ошибки должны логироваться.
- Если ИИ не уверен в тексте или позиции, использовать статус `needs_review`.
- Если цена неизвестна, использовать статус позиции `needs_price`.
- Нельзя удалять записи физически; спорные данные переводить в статус.

## Перед завершением работы

Перед финальным ответом запустить:

- `npm run lint`
- `npm run test`
- `npm run build`

Если команда не прошла, описать причину, исправить проблему и повторить проверку.
