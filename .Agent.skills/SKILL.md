# Рабочий skill

## Обязательный порядок

1. Прочитать `AGENTS.md`.
2. Проверить `git status -sb`, `git diff --stat`, `git diff`.
3. Если работа начинается с `main`, создать ветку от `origin/main`: `agent/<kratkoe-opisanie>`.
4. Сделать минимальную правку без несвязанных изменений.
5. Обновить документацию и `CHANGELOG.md`, если меняется поведение, структура, проверки или правила.
6. Запустить проверки из `AGENTS.md`.
7. Закоммитить только файлы задачи.
8. Отправить ветку на GitHub.
9. Создать draft PR в `main`, если доступен авторизованный `gh`.

## Запреты

1. Не смешивать разные задачи в одной ветке.
2. Не пушить напрямую в `main` без прямой просьбы.
3. Не обходить проблемы доступа к GitHub.
4. Не оставлять финальный ответ без ветки, commit, проверок и статуса push/PR.
