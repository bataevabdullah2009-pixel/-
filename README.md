# Голосовой журнал продаж

Telegram-бот + Telegram WebApp для магазина, где продавец надиктовывает продажи голосом, а система сохраняет распознанные товары, цены, количество и выручку в Supabase.

## Текущая модель продукта

```text
Продавец -> голосовое сообщение в Telegram
  -> бот скачивает аудио
  -> аудио подготавливается для STT
  -> STT возвращает русскую расшифровку
  -> LLM/парсер возвращает позиции продажи
  -> детерминированный резервный разбор разделяет склеенные или частично распознанные позиции
  -> Supabase сохраняет voice_records, sales, sale_items
  -> Telegram отправляет сообщение об успехе или кнопки решения по проверке
  -> WebApp показывает отчёт, записи и продавцов
```

Уверенная запись получает `processed`, сразу входит в выручку и получает обычный ответ бота без inline-кнопок.

Сомнительная запись получает `needs_review` и подтверждается inline-кнопками Telegram `✅ Подтвердить` / `❌ Отмена` либо вкладкой WebApp `Проверка`. Неполные позиции не входят в выручку; активные позиции `processed` учитываются даже если родительская продажа остаётся `needs_review` из-за других неполных строк. В сообщении сомнительной записи нет кнопки `Открыть отчёт`.

Подтверждение работает на уровне `sale_items`: если в корзине есть хотя бы одна полная позиция, она переводится в `processed`, попадает в `sales.total_amount` и начинает входить в выручку. Неполные позиции той же записи остаются активными `needs_review` и продолжают отображаться во вкладке `Проверка`. Если нет ни одной полной позиции, подтверждение не выполняется и пользователь видит сообщение `Не удалось подтвердить: нет ни одной полной позиции.` При успехе Telegram/WebApp показывают `✅ Подтверждено: N позиций, сумма X ₽`.

Резервный парсер поддерживает фразы с точкой, запятой после названия, союзом `и`, новой строкой и несколькими товарами подряд. Если LLM вернул одну длинную позицию вроде `Сникерс, 3 штуки по 200 рублей. Буханка хлеба, 5 штук по 50 рублей.`, резервный разбор создаёт две строки `sale_items`. Неполные остатки, например `Корзина продуктов`, сохраняются отдельной позицией на проверку и не блокируют валидные товары.

Отмена переводит запись в `cancelled`, мягко удаляет активные позиции и оставляет запись вне выручки. `failed` используется для сбоев голосового конвейера и не входит в отчёт.

## WebApp

Нижняя навигация содержит четыре раздела:

- `Отчёт`;
- `Проверка`;
- `Записи`;
- `Продавцы`.

Экран `Отчёт` использует спокойную премиальную SaaS-панель:

- фон `#0B1020`;
- поверхности `#12192B` и `#161F34`;
- основной акцент `#5B8CFF`;
- предупреждение `#F59E0B` только для состояний проверки;
- опасность `#EF4444` только для удаления и ошибок.

Экран показывает:

- один заголовок `Сводка магазина`;
- название магазина и выбранный период;
- карточки `Выручка`, `Количество товаров`, `Записей`, `Нужно проверить`;
- компактную аналитику по дням с суммами;
- компактный выбор периода: сегодня, вчера, неделя, месяц и дата;
- топ товаров;
- продажи за период;
- блок `Нужно проверить`, если есть активные данные `needs_review`, без смешивания с выручкой.

Экран `Проверка` показывает только активные позиции `needs_review`. Каждая позиция остаётся отдельной карточкой, может быть отредактирована или исключена, а кнопки `Подтвердить`, `Отмена` и `Подтвердить всё` вызывают серверные действия для родительской продажи. Эти действия синхронизируются с теми же статусами и выручкой, что callback Telegram.

Экран `Записи` показывает журнал голосовых продаж: дата, продавец, распознанный текст, статус, сумма, аудио при наличии и раскрытие товаров. Для `needs_review` показывается бейдж `Нужно подтвердить в Telegram`.

Экран `Продавцы` показывает имя, активность, количество записей и выручку за выбранный период.

## Редактирование товаров

Карточка товара в WebApp имеет компактный обычный режим:

```text
Сникерс
5 шт × 100 ₽
500 ₽                         ✏️ 🗑
```

Карандаш открывает компактный режим редактирования с полями `Товар`, `Количество`, `Единица`, `Цена, ₽`, кнопками `Сохранить` и `Отмена`. Единица поддерживает `шт`, `кг` и `г`; парсер также нормализует `бутылка/бутылки/бутылок` в `шт`. Для граммов сумма считается как доля килограмма от цены за кг.

Сохранение вручную реально обновляет строку `sale_items`, пересчитывает `total`, ставит позицию в `processed` и убирает ошибку цены. Если родительская продажа ещё `needs_review`, эта позиция уже может войти в выручку; неполные соседние позиции остаются в `Проверке`.

Корзина открывает локальное окно подтверждения `Удалить товар из отчёта?` с действиями `Удалить` и `Отмена`.

Сохранение и удаление выполняются через серверные действия, которые используют административный клиент Supabase, проверяют доступ к магазину через сессию Telegram WebApp и пересчитывают `sales.total_amount`.

## Выручка

В выручку входят только позиции, которые одновременно удовлетворяют условиям:

- `sale_items.status = processed`;
- родительский `sales.status` не `cancelled` и не `failed`;
- `sale_items.deleted_at is null`;
- `sale_items.total is not null`;
- количество/вес валидны;
- `sale_items.price` валиден либо может быть восстановлен из `total / quantity`.

Не входят:

- `needs_review`;
- parent sale `cancelled`;
- parent sale `failed`;
- `excluded`;
- мягко удалённые строки;
- активные позиции со статусами `needs_review`, `needs_price` или `failed`.

## Telegram-бот

Основные команды и сценарии:

- `/start` регистрирует или проверяет продавца, выдаёт кнопку отчёта WebApp и постоянную кнопку меню;
- голосовое сообщение запускает конвейер распознавания;
- уверенная запись отвечает `✅ Запись сохранена: ...`;
- сомнительная запись отвечает `⚠️ Запись сохранена, но нужно подтвердить товары и цены. Распознано: ...`;
- `✅ Подтвердить` переводит sale/voice и валидные active items в `processed`;
- успешное подтверждение отвечает `✅ Подтверждено: N позиций, сумма X ₽`;
- неполные активные позиции той же записи остаются `needs_review` и не входят в выручку;
- `❌ Отмена` переводит продажу/голосовую запись в `cancelled` и мягко удаляет активные позиции;
- webhook должен быть установлен с `allowed_updates: ["message", "callback_query"]`.

Данные callback:

```text
confirm:<sale_id>
cancel:<sale_id>
```

Устаревший префикс callback `voice_sale_review:` принимается для уже отправленных старых сообщений, но новые сообщения используют короткий формат.

## Окружение

Локальная разработка использует `.env.local`; шаблон лежит в `.env.example`.

- `TELEGRAM_BOT_TOKEN` - токен Telegram-бота.
- `TELEGRAM_WEBHOOK_SECRET` - секрет маршрута webhook.
- `NEXT_PUBLIC_APP_URL` - публичный URL WebApp.
- `PUBLIC_WEBHOOK_URL` - опциональный отдельный URL webhook или полный `/api/telegram/webhook`.
- `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY` - ключи проекта Supabase; service role используется только на сервере.
- `SUPABASE_STORAGE_BUCKET` - bucket для голосового аудио, по умолчанию `voice-records`.
- `STT_API_KEY`, `STT_API_URL`, `STT_MODEL` - speech-to-text.
- `LLM_API_KEY`, `LLM_API_URL`, `LLM_MODEL` - parser/cleanup LLM.
- `DEMO_MODE`, `DEMO_OWNER_TELEGRAM_ID`, `DEFAULT_SHOP_NAME` - демо/bootstrap-режим.
- `ALLOW_WEBAPP_FALLBACK`, `DEFAULT_SHOP_ID`, `DEFAULT_SELLER_ID` - локальный резервный режим WebApp без Telegram initData.
- `DEBUG_TELEGRAM_WEBAPP` - диагностический экран Telegram WebApp.

## Локальный запуск

1. Установить зависимости: `npm install`.
2. Заполнить `.env.local`.
3. Запустить bot + WebApp: `npm run dev`.
4. Для отдельного WebApp: `npm run web:dev`.
5. Если нужен Telegram webhook, задать публичный HTTPS URL и выполнить `npm run telegram:set-webhook`.

## Развертывание

WebApp и webhook рассчитаны на Vercel/Next.js App Router. Перед развертыванием проверить переменные окружения, миграции Supabase, Storage bucket, URL webhook и `allowed_updates: ["message", "callback_query"]`. После развертывания выполнить `npm run telegram:webhook-info`, отправить тестовое голосовое сообщение и проверить логи Vercel/Supabase без секретов.

## Проверка продукта

Проверять перед сдачей:

1. `Буханка хлеба пять штук по сто рублей. Сникерс три штуки по двести рублей.` создаёт 2 позиции, сумма `1100 ₽`, статус `processed`, без ошибки подтверждения.
2. `Буханка хлеба, 6 штук по 200 рублей. Масса сливочная, 1 штука по 100 рублей.` попадает в `Проверка`, если уверенность требует проверки; подтверждение не показывает ложную ошибку, валидные позиции входят в выручку.
3. `Корзина продуктов.` не подтверждается и возвращает `Не удалось подтвердить: нет ни одной полной позиции.`
4. Повторный `✅ Подтвердить` по уже подтверждённой записи возвращает `✅ Уже подтверждено`, без дублей и ошибок.
5. `❌ Отмена` переводит запись в `cancelled`, мягко удаляет активные позиции, исключает их из выручки и обновляет WebApp.
6. `✏️` редактирует товар, сохраняет строку Supabase, пересчитывает сумму позиции и итоги отчёта.
7. `🗑` мягко удаляет товар, он исчезает из активного отчёта и не возвращается после перезагрузки.
