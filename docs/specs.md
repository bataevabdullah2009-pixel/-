# Техническое задание

## Цель

Создать простую систему голосовой фиксации продаж для магазина.

Продавец говорит продажи голосом, система превращает голос в текст, сохраняет запись и формирует простые отчёты по товарам, количеству и выручке.

## Пользователи

- Владелец магазина.
- Продавец.
- Преподаватель, который проверяет учебный проект.

## Главная функция

Продавец отправляет голосовое сообщение в Telegram. Система:

1. Скачивает аудио.
2. Сохраняет аудио в Supabase Storage.
3. Распознаёт речь в текст.
4. Очищает текст.
5. Извлекает позиции продажи в строгом JSON.
6. Сохраняет исходную запись, распознанный текст, продажу и позиции.
7. Показывает владельцу записи и отчёты.

## Поток обработки голосового

1. Продавец отправляет голосовое.
2. STT переводит голос в текст.
3. ИИ очищает текст.
4. ИИ извлекает структурированные позиции:
   - `product_name`
   - `quantity`
   - `unit`
   - `price`
   - `total`
   - `confidence`
5. Если цена названа в голосе, использовать эту цену.
6. Если цена не названа, попытаться найти цену в таблице `products`.
7. Если цена не найдена, сохранить позицию со статусом `needs_price`.
8. Сохранить исходную голосовую запись.
9. Сохранить распознанный текст.
10. Сохранить позиции продажи.
11. В веб-панели показать дневной итог в таблице.

## Строгий JSON от ИИ-парсера

```json
{
  "items": [
    {
      "product_name": "Хлеб",
      "quantity": 3,
      "unit": "шт",
      "price": 40,
      "total": 120,
      "confidence": 0.95
    }
  ],
  "raw_text": "хлеб 3 по 40",
  "cleaned_text": "Хлеб — 3 штуки по 40 рублей.",
  "needs_review": false
}
```

Если цена не названа:

```json
{
  "items": [
    {
      "product_name": "Хлеб",
      "quantity": 3,
      "unit": "шт",
      "price": null,
      "total": null,
      "confidence": 0.9
    }
  ],
  "raw_text": "хлеб 3 штуки",
  "cleaned_text": "Хлеб — 3 штуки.",
  "needs_review": true
}
```

## Что входит в MVP

- Telegram-бот.
- Команда `/start`.
- Приём голосовых сообщений.
- Скачивание аудио из Telegram.
- Хранение аудио в Supabase Storage.
- Распознавание речи в текст.
- Очистка текста через ИИ.
- Извлечение товарных позиций из текста.
- Сохранение голосовой записи в `voice_records`.
- Сохранение продажи в `sales`.
- Сохранение позиций в `sale_items`.
- Таблица `products` с ценой по умолчанию.
- Расчёт `total` для каждой позиции, если цена известна.
- Веб-панель с записями.
- Фильтры по дате.
- Фильтр по продавцу.
- Поиск по тексту.
- Страница `/daily-report`.
- Дневной, недельный, месячный и годовой отчёты.
- Группировка одинаковых товаров.
- Таблица отчёта: товар, количество, выручка.
- Итоговая строка.
- Блок “Нужно проверить”.
- Ручное исправление спорной позиции через веб-форму.
- Статусы обработки голосовой записи:
  - `pending`
  - `processed`
  - `needs_review`
  - `failed`
- Статусы позиций:
  - `processed`
  - `needs_price`
  - `needs_review`
  - `failed`

## Правила расчёта

- ИИ не должен выдумывать цену.
- ИИ не должен выдумывать товар.
- Если количество не названо, считать `quantity = 1`, но снижать `confidence`.
- Если цена неизвестна, не считать выручку по этой позиции.
- В итоговую выручку попадают только позиции с известной ценой и статусом `processed`.
- Позиции без цены попадают в блок “Нужно проверить”.
- Спорные записи должны быть доступны для ручного исправления.

## Веб-панель

### `/records`

Показывает список голосовых записей и продаж:

- дата;
- время;
- продавец;
- очищенный текст;
- статус;
- кнопка прослушать аудио, если `audio_url` есть.

Фильтры:

- Сегодня.
- Неделя.
- Месяц.
- Год.
- Произвольная дата.
- Продавец.
- Поиск.

### `/daily-report`

Показывает простой отчёт:

- заголовок “Отчёт за день”;
- выбор даты;
- фильтры: сегодня, вчера, неделя, месяц, год, ручной выбор даты;
- таблица:
  - товар;
  - количество;
  - выручка;
- строка “Итого”;
- блок “Нужно проверить” для позиций без цены или с низкой уверенностью.

В дневном отчёте по умолчанию не показываются продавец, время, длинные сырые записи и технические статусы.
