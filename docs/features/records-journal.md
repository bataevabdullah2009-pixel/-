# Журнал записей

Журнал `Записи` показывает историю голосовых продаж текущего магазина, определённого на сервере. Он нужен для проверки исходного распознавания, аудио и состава товаров, но не является основным экраном решения проверки.

## Источник

`getRecords(filters)`:

1. Вызывает `requireOwner()`.
2. Получает текущий магазин на сервере.
3. Читает `sales` по магазину и периоду.
4. Применяет опциональный фильтр продавца.
5. Применяет текстовый поиск по `raw_text` и `cleaned_text`.
6. Читает связанные `sale_items`.
7. Читает имя связанного продавца.
8. Читает связанные поля голосового аудио.
9. Создаёт signed URL аудио, если есть `audio_path`.
10. Возвращает записи и ошибку отдельно.

Фильтр продавца не расширяет границу магазина.

## Карточка записи

Карточка содержит:

1. Дату.
2. Время.
3. Продавца.
4. Распознанный текст.
5. Резервный raw-текст.
6. Пользовательский статус.
7. Сумму.
8. Ссылку на аудио, если аудио сохранено.
9. Раскрытие `Товары`.
10. Список позиций продажи.

## Статусы

Внутренние enum не показываются напрямую.

Подписи:

1. `processed` -> `Готово`.
2. `needs_review` -> `Нужно проверить`.
3. Устаревший `needs_price` -> `Нужно проверить`.
4. `failed` -> `Нужно проверить`.
5. `pending` -> `Нужно проверить`.
6. `cancelled` -> `Исключено`.
7. `excluded` -> `Исключено`.

Для `needs_review` показывается badge `Нужно проверить`.

## Товары внутри записи

Раскрытие товаров показывает:

1. Название товара.
2. Количество.
3. Единицу.
4. Цену.
5. Сумму.
6. Статус позиции.
7. Состояние удаления, если позиция исключена.

Журнал может показать позиции родительской продажи, включая контекст проверки и удаления, но активная выручка всё равно считается на экране отчёта.

## Что журнал не делает

1. Не подтверждает продажу на проверке.
2. Не отменяет продажу на проверке.
3. Не принимает доверенный `shop_id`.
4. Не пишет напрямую в Supabase из браузера.
5. Не скрывает ошибку auth/БД как пустое состояние.

Подтверждение/отмена выполняются в callback Telegram или на вкладке `Проверка`.

## Аудио

Аудио:

1. Загружается best-effort во время голосового конвейера.
2. Отсутствие аудио не ломает запись.
3. `audio_path` даёт подписанный URL через сервер.
4. `audio_url` используется как fallback.
5. Подписанный URL живёт ограниченное время.
6. Ошибки Storage не раскрываются пользователю как raw-ошибка Supabase.

## Фильтры

Поддерживаются:

1. Период.
2. Пользовательская дата.
3. Продавец.
4. Поиск.

Поиск ищет по:

1. `raw_text`.
2. `cleaned_text`.

## Пустые состояния и ошибки

1. Если записей нет и ошибки нет, показывается пустое состояние.
2. Если ошибка auth, показывается уведомление действия.
3. Если ошибка БД, показывается уведомление действия.
4. Ошибка не должна выглядеть как `Записей нет`.

## Критерии приёмки

1. Записи используют магазин, определённый на сервере.
2. Фильтр продавца не может раскрыть другой магазин.
3. Поиск не может раскрыть другой магазин.
4. Карточка записи показывает контекст расшифровки.
5. Ссылка на аудио появляется только при наличии.
6. Подписи внутренних enum переведены.
7. Решение проверки не дублируется в журнале.
8. Ошибки auth/БД остаются видимыми.
