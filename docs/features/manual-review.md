# Ручная проверка товаров

Блок «Записанные товары» показывает активные позиции выбранного периода. Позиции `needs_price`, `needs_review` и `failed` отмечаются для проверки; обработанные позиции также можно исправить.

## Сохранение

Владелец меняет `product_name`, `quantity` и `price`. Серверная функция `updateSaleItem`:

1. проверяет непустое имя, `quantity > 0` и `price > 0`;
2. загружает ровно одну активную позицию по `sale_items.id`;
3. сохраняет существующую единицу или единицу совпавшего товара;
4. пересчитывает `total = quantity × price`;
5. ставит `status = processed`, `confidence = 1` и `updated_at = now()`;
6. пересчитывает родительскую продажу, пишет аудит и обновляет страницы отчёта.

После успеха позиция исчезает из блока проверки. При отказе показывается только: «Не удалось сохранить товар. Проверьте данные и попробуйте ещё раз.»

## Исключение

`excludeSaleItem` не выполняет SQL `DELETE`. Позиция получает `deleted_at`, `deleted_reason = excluded_by_owner`, `status = excluded`, `updated_at` и сохранённый `deleted_previous_status`. Она исчезает из количества и выручки, но остаётся в панели исключённых и может быть восстановлена.

При отказе показывается только: «Не удалось исключить товар из отчёта.» Техническая ошибка остаётся на сервере.

## Ограничения

Мутации используют service role только в server-only модулях. Производственная авторизация владельца и транзакционность изменения, пересчёта и аудита пока не реализованы.
