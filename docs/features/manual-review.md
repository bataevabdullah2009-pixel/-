# Ручное исправление

Владелец изменяет название, количество и цену позиции. Сервер требует непустое осмысленное название, `quantity > 0` и `price > 0`, обновляет строку по id, пересчитывает total, выставляет `processed`, `confidence = 1`, `updated_at = now()` и обновляет связанную продажу.

После revalidation позиция исчезает из блока ручного разбора и появляется в активной сводке.
