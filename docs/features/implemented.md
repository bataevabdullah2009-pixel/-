# Реализованные функции

Краткий список фактически реализованных возможностей. Подробные системные правила статусов, проверки, подтверждения, отмены и выручки описаны в [../specs/00-global.md](../specs/00-global.md).

## Telegram-бот

1. `/start` проверяет продавца и даёт доступ к WebApp.
2. Голосовые сообщения проходят скачивание, подготовку аудио, STT, parser cleanup и сохранение.
3. Детерминированный резервный разбор разделяет склеенные фразы на отдельные `sale_items`.
4. Уверенные продажи сохраняются как `processed`.
5. Сомнительные продажи сохраняются как `needs_review`.
6. Сообщение проверки содержит только `✅ Подтвердить` и `❌ Отмена`.
7. Callback использует короткие данные `confirm:<sale_id>` и `cancel:<sale_id>`.
8. Устаревший callback-префикс принимается для старых сообщений.
9. Подтверждение и отмена идемпотентны.
10. Сбой голосовой обработки фиксируется диагностируемым статусом, если продажа не сохранена.

## WebApp

1. Нижняя навигация содержит `Отчёт`, `Проверка`, `Записи`, `Продавцы`.
2. `Отчёт` показывает выручку, количество товаров, количество записей, проверки, топ товаров и продажи за период.
3. `Проверка` показывает активные позиции `needs_review`, подтверждение, отмену и массовое подтверждение.
4. `Записи` показывает журнал голосовых продаж, распознанный текст, статус, сумму, товары и аудио при наличии.
5. `Записи` поддерживает поиск по тексту и фильтр продавца.
6. `Продавцы` показывает активность, количество записей, последнюю активность и выручку за выбранный период.
7. `/debug-telegram` закрыт для production без `DEBUG_TELEGRAM_WEBAPP=true`.

## Управление позициями

1. Карандаш открывает компактное редактирование позиции.
2. Валидное сохранение обновляет `sale_items`, пересчитывает сумму и ставит позицию `processed`.
3. Родительская продажа может оставаться `needs_review`, если рядом есть неполные позиции.
4. Корзина открывает подтверждение удаления.
5. Удаление выполняет soft delete через `deleted_at`.
6. Удалённая позиция исчезает из активного отчёта и не возвращается после перезагрузки.
7. Восстановление возвращает мягко удалённую позицию с предыдущим статусом и пересчётом продажи.
8. Сброс дня мягко удаляет активные позиции выбранного дня без физического удаления `sales`.

## Данные и выручка

1. Supabase хранит магазины, продавцов, каталог товаров, голосовые записи, продажи и позиции.
2. `save_voice_sale` сохраняет голосовую запись, продажу и позиции.
3. Отчёт ограничен текущим магазином.
4. Выручка считается по активным `sale_items.status = processed`, если родительская продажа не `cancelled` и не `failed`.
5. `needs_review`, `needs_price`, `failed`, `excluded` и мягко удалённые позиции не входят в выручку.
