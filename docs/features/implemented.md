# Реализованные функции

## Telegram-бот

- `/start` проверяет продавца и даёт WebApp доступ к отчёту.
- Голосовые сообщения проходят скачивание, подготовку аудио, STT, парсер и сохранение в Supabase.
- Детерминированный резервный разбор разделяет склеенную расшифровку с несколькими позициями на отдельные `sale_items` и сохраняет неполные остатки отдельными строками проверки.
- Уверенные продажи сохраняются как `processed`.
- Сомнительные продажи сохраняются как `needs_review`.
- Сообщение проверки содержит только `✅ Подтвердить` и `❌ Отмена`.
- Данные callback короткие: `confirm:<sale_id>` и `cancel:<sale_id>`.
- Устаревший префикс callback принимается для старых сообщений.
- Callback подтверждения переводит продажу/голосовую запись и валидные активные позиции в `processed`; неполные активные позиции смешанной корзины остаются `needs_review`.
- Callback отмены переводит продажу/голосовую запись в `cancelled` и мягко удаляет активные позиции.
- Сбой голосовой обработки сохраняет `voice_records.status = failed`, если продажа ещё не сохранена.

## WebApp

- Нижняя навигация: `Отчёт`, `Проверка`, `Записи`, `Продавцы`.
- `Отчёт` показывает выручку, количество товаров, количество записей и количество проверок.
- Период фильтруется по сегодня, вчера, неделя, месяц и выбранная дата.
- `Топ товаров` строится по активной обработанной выручке.
- `Продажи за период` показывает карточки активных обработанных позиций.
- `Проверка` показывает активные позиции проверки и даёт `Подтвердить`, `Отмена`, `Подтвердить всё`.
- `Записи` показывает журнал голосовых продаж с распознанным текстом, статусом, суммой, ссылкой на аудио и раскрытием товаров.
- Запись `needs_review` помечается бейджем проверки и доступна на вкладке `Проверка`.
- `Продавцы` показывает активность, количество записей и выручку за выбранный период.
- `/debug-telegram` доступен только в разработке или при `DEBUG_TELEGRAM_WEBAPP=true`.

## Управление позициями продажи

- `✏️` открывает компактную форму редактирования.
- Валидное ручное сохранение обновляет `sale_items`, пересчитывает `total` и ставит позицию `processed`; родительская продажа `needs_review` может оставаться на проверке для других неполных позиций.
- Редактирование сохраняет товар, количество, единицу и цену в Supabase.
- Редактирование пересчитывает сумму позиции.
- Редактирование пересчитывает сумму продажи и итоги отчёта.
- `🗑` открывает окно подтверждения `Удалить товар из отчёта?`.
- Удаление выполняет мягкое удаление через `deleted_at`.
- Удалённая позиция исчезает из активного отчёта.
- Удалённая позиция не возвращается после перезагрузки страницы.
- Восстановление остаётся для исторически мягко удалённых строк в отдельном блоке деталей.

## Правила выручки

- В выручку входит активная позиция `processed`, если родительская продажа не `cancelled` и не `failed`.
- Позиция с валидным `total` может войти в выручку, даже если цена за единицу была восстановлена из total.
- Позиции `needs_review`/`needs_price`/`failed`/`excluded`, родительские `cancelled`/`failed` и удалённые строки не входят.
- Обработанная позиция внутри продажи `needs_review` входит в выручку; неполные соседние позиции остаются в проверке.
