# Отчёт продаж

Telegram-кнопка «Открыть отчёт» является Web App button. После Telegram auth отчёт показывает только текущий server-derived shop.

Основная сводка содержит только активные `processed` items. `needs_review` и `needs_price` всегда отображаются отдельно в «Нужно проверить» с формой исправления. Ошибка авторизации показывается сообщением, а не нулевым успешным отчётом.

Save, «Исключить из отчёта» и «Сбросить день» выполняются Server Actions с service role только на сервере. После операции sale пересчитывается, пути `/daily-report` и `/records` revalidate, поэтому ручная перезагрузка не нужна.
