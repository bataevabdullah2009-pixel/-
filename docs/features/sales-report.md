# Sales Report

Отчёт показывает выручку и количество за выбранный период.

В итог входят только активные готовые позиции (`processed`; legacy `confirmed` также распознаётся расчётчиком). Review и excluded позиции не учитываются.

Review-позиции всегда выводятся отдельно в «Нужно проверить». Ошибка Telegram auth или Supabase не маскируется под пустой отчёт и не показывает успешные нулевые метрики.

Чтение выполняется в два шага: `sales` фильтруются по server-derived `shop_id` и периоду, затем `sale_items` читаются только по найденным sale IDs. Report log фиксирует Telegram user, seller, shop, counts, период и безопасную причину ошибки.

Периоды считаются по `Europe/Moscow` с полуинтервалом `[start, end)`.

На странице есть:

- summary bar с периодом, выручкой и количеством;
- компактные карточки товаров с количеством, ценой за единицу и суммой;
- блок «Нужно проверить»;
- блок «Готовые продажи»;
- список исключённых товаров с восстановлением;
- действия refresh и reset day.

Редактирование и удаление находятся справа в карточке. Update/delete возвращают локальный action state, поэтому loading и ошибки не требуют полного redirect. Если активных items нет, UI показывает «Нет активных товаров за выбранный период», даже если родительская `sale` сохранена.
