# Отчёт о продажах

`getReport` строит отчёт по `sale_items`, связанным с продажами выбранного календарного диапазона `Europe/Moscow`. В итог входят только строки с `deleted_at is null`, `status = processed` и заполненными `price`/`total`.

Статусы `needs_price`, `needs_review`, `failed` и `excluded` не увеличивают количество или выручку. Мягко удалённые строки не учитываются независимо от сохранённого предыдущего статуса.

Допустимые позиции группируются по `product_id`, а при его отсутствии — по нормализованному `product_name`. Общая выручка равна сумме `total`, общее количество — сумме `quantity` обработанных активных позиций. После сохранения, исключения, восстановления или сброса `sales.total_amount` пересчитывается по тем же правилам.

`getReviewItems` возвращает активные позиции блока проверки. Ручное исправление переводит корректную позицию в `processed`; ручное исключение переводит её в `excluded` и заполняет метаданные soft delete.

Текущее известное ограничение: общее количество суммирует разные единицы (`шт`, `кг`) одним числом.
