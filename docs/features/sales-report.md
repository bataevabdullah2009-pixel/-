# Отчёт продаж

Все Telegram-кнопки «Открыть отчёт» являются Web App buttons. После Telegram auth отчёт показывает только текущий server-derived shop; переданный в URL `shop_id` игнорируется.

Основная сводка и блок «Подтверждённые продажи» содержат только активные `processed` items. Все новые voice items отображаются в «Нужно проверить» с формой исправления и отдельным подтверждением. Ошибка авторизации показывается сообщением, а не нулевым успешным отчётом.

Save, confirm, «Исключить из отчёта», restore и «Сбросить день» выполняются Server Actions с service role только на сервере. После операции sale пересчитывается, пути `/daily-report` и `/records` revalidate, поэтому полная перезагрузка документа не нужна.
