# Планы

Планы разделены по состоянию, а каждая существенная задача имеет проверяемый результат. В `active` одновременно допускается только один предметный план.

| Раздел | Назначение |
| --- | --- |
| [Активные](./active/README.md) | Текущая работа: план, контрольный список, приёмка и заметки. |
| [Завершённые](./completed/README.md) | История реализованных этапов и доказательства проверки. |
| [Будущие](./backlog/README.md) | Идеи после MVP, не входящие в текущую сдачу. |

Жизненный цикл: выбрать один backlog-пункт, перенести его в `active`, выполнить контрольный список, зафиксировать проверку и перенести результат в `completed`. Не разбивать последовательные этапы на несколько параллельных активных планов.
