# Monitoring и alerts

## Цель

Обнаруживать отказы webhook, STT, LLM и Supabase до потери наблюдаемости продаж.

## Результат

- correlation ID;
- метрики успешности и latency;
- error tracking без секретов;
- alerts по `failed`, очереди и внешним сервисам;
- runbook и проверка оповещений.

Детали: `docs/specs/technical/monitoring.md`.
