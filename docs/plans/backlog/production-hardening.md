# Production hardening

## Цель

Сделать обработку Telegram update и запись продаж устойчивыми до публичного тестирования.

## Объём

- идемпотентность и уникальный ключ update;
- транзакционная запись `sale` + `sale_items`;
- явная state machine ошибок и повторов;
- интеграционные тесты webhook и Supabase;
- формальная политика хранения аудио.

## Условие активации

План `003-curator-release` завершён. При активации создаётся один plan folder; соседние backlog-пункты не становятся активными автоматически.
