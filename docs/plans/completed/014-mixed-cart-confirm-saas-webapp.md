# 014 - Подтверждение смешанной корзины и спокойный SaaS WebApp

Дата: 2026-07-02

## Цель

Исправить сценарий подтверждения после голосового распознавания так, чтобы одна неполная позиция не блокировала всю корзину, и заменить резкий оранжево-чёрный WebApp на спокойный премиальный SaaS-интерфейс.

## Сценарий подтверждения

- Причина: прежнее подтверждение требовало, чтобы все активные `sale_items` были полными. Если парсер создавал одну валидную позицию и одну неполную строку, handler считал всю корзину невалидной.
- Telegram `confirmVoiceSaleWithClient` и WebApp `confirmReviewSale` теперь работают на уровне `sale_items`.
- Валидная позиция: осмысленный `product_name`, положительное `quantity` или весовая величина, а также сохранённый `price` или `total`, из которого можно вычислить цену за единицу.
- Валидные позиции получают `processed`, `confidence = 1`, пересчитанные `price` и `total`.
- Неполные активные позиции остаются в `needs_review` и не входят в выручку.
- Если нет ни одной полной позиции, подтверждение не меняет продажу/позиции и возвращает `Не удалось подтвердить: нет ни одной полной позиции.`

## Единицы и суммы

- `шт` и `кг` считаются напрямую.
- `г`, `гр`, `грамм`, `грамма`, `граммов` нормализуются в `г`.
- Для граммов сумма считается как доля килограмма: `300 г по 200 ₽ = 60 ₽`.
- Если есть сумма и количество/вес, но нет цены за единицу, цена за единицу вычисляется из суммы.

## WebApp

- Основная палитра: фон `#0B1020`, поверхность `#12192B`, вторая поверхность `#161F34`, граница `rgba(255,255,255,0.08)`, текст `#F3F6FB`, приглушённый `#94A3B8`, акцент `#5B8CFF`.
- Оранжевый больше не является основным цветом интерфейса и используется только как предупреждение.
- Header стал компактным: один заголовок и контекст магазина.
- KPI остаётся сеткой 2x2.
- Фильтры периода: `Сегодня`, `Вчера`, `Неделя`, `Месяц` и выбор даты.
- Топ товаров ограничен пятью строками, длинные названия обрезаются.
- Последние продажи используют компактные карточки с редактированием/удалением справа.
- `Проверка` показывает причины неполноты: нет цены, нет количества/веса, не удалось выделить отдельный товар.
- Режим редактирования поддерживает товар, количество, единицу и цену.
- Metadata WebApp указывает `/favicon.svg` в новой палитре, чтобы browser smoke не ловил 404 на отсутствующей иконке.

## Проверка

Release gate для этой работы:

- `npm.cmd run lint`
- `npm.cmd run test`
- `npm.cmd run build`
- browser smoke WebApp

Отдельные regression tests покрывают:

- полную корзину `Сникерс 5 штук по 100 рублей. Буханка хлеба 3 штуки по 50 рублей`;
- смешанную корзину с валидной позицией и неполной строкой;
- корзину без полной позиции.
