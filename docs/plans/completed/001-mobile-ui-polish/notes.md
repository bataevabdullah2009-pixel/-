# Заметки

- Используется мягкое удаление через `sale_items.deleted_at`, а не физический `DELETE`.
- `deleted_reason` различает ручное исключение `excluded_by_owner`, сброс `day_reset` и прежние строки `manual`.
- Ручное исключение ставит отдельный статус `excluded`; восстановление возвращает `deleted_previous_status`.
- `updated_at` меняется при сохранении, исключении, восстановлении и сбросе.
- Итог и статус продажи выводятся из активных позиций.
- Исходная голосовая запись и JSON парсера сохраняются.
- Миграция применяется до развёртывания зависимого кода.
- Авторизация владельца пока отсутствует; серверные действия допустимы только для учебных данных.
- Локальная проверка 18 июня 2026 года: 390 px и компьютер без переполнения и ошибок консоли; действия с подключёнными данными не выполнялись.
- План архивирован как локально завершённый, чтобы не держать несколько параллельных активных этапов.
- Регрессионная проверка 19 июня 2026 года покрывает `20 × 300 = 6000`, статус `excluded` и исключение неактивных статусов из выручки.
