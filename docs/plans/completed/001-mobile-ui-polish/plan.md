# План: улучшение мобильного интерфейса

Статус: завершено локально; внешняя демонстрация включена в `003-curator-release`.

## Цель

Сделать Telegram Web App быстрым и понятным на телефоне: владелец видит итоги, редактирует или исключает товар, восстанавливает ошибочно исключённую позицию и сбрасывает выбранный день.

## Проблема

Исходные элементы были мелкими и неоднородными, редактирование было доступно только сомнительным позициям, исключения и сброса дня не было.

## Объём работ

- обновить визуальную систему, навигацию, карточки, формы и опасные действия;
- разрешить правку любой активной позиции;
- реализовать мягкое удаление, восстановление и сброс одного дня;
- стабилизировать server actions сохранения и исключения: update по `sale_items.id`, `updated_at`, `excluded` и безопасные сообщения ошибок;
- показывать результат действий и требовать подтверждения;
- сохранить таблицу на компьютере и доступность.

## Затрагиваемые области

`apps/web`, `packages/shared`, `tests`, новая миграция Supabase и связанные спецификации.

## Ограничения

- не менять webhook, STT, LLM-парсер и режимы Telegram;
- не менять `.env.local`;
- не удалять физически голос, продажу или аудио.

## Порядок

1. Добавить модель мягкого удаления миграцией.
2. Реализовать серверные мутации и пересчёт.
3. Обновить интерфейс и подтверждения.
4. Добавить тесты и документацию.
5. Выполнить команды качества и мобильную проверку.
