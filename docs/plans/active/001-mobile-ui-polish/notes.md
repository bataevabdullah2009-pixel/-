# Заметки

- Используется мягкое удаление через `sale_items.deleted_at`, а не физический `DELETE`.
- `deleted_reason` различает ручное исключение и сброс дня.
- Итог и статус продажи выводятся из активных позиций.
- Исходная голосовая запись и JSON парсера сохраняются.
- Миграция применяется до развёртывания зависимого кода.
- Авторизация владельца пока отсутствует; серверные действия допустимы только для учебных данных.
- Локальная проверка 18 июня 2026 года: 390 px и компьютер без переполнения и ошибок консоли; действия с подключёнными данными не выполнялись.
