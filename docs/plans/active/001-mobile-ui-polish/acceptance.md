# Приёмка

## Критерии готовности

- владелец изменяет название, количество и цену активной позиции;
- исключённая позиция сразу исчезает из итогов, но остаётся в БД;
- восстановление возвращает позицию и пересчитывает выручку;
- сброс доступен только для одного дня и обратим;
- интерфейс не имеет горизонтальной прокрутки на ширине 360 px;
- webhook, бот, STT и парсер не изменены.

## Проверка

1. Открыть `/daily-report?period=today`.
2. Изменить цену и проверить итог.
3. Исключить и восстановить позицию.
4. Сбросить день и восстановить одну позицию.
5. Выполнить `npm run lint`, `npm run test`, `npm run build`.

Работа считается полностью принятой только после применения миграции и проверки на целевом демонстрационном Supabase.
