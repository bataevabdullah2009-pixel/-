# Приёмка

Приёмка относится только к демонстрационному MVP и не означает production-readiness.

## Критерии готовности

- рабочий webhook отвечает без `last_error_message`;
- один update создаёт одну продажу в проверяемом сценарии;
- запись и отчёт появляются в Supabase и Web App;
- мутации пересчитывают продажу и создают аудит;
- куратор повторяет запуск по описанию и спецификации развёртывания.

## Проверка

Выполнить команды из `docs/specs/technical/deployment-vercel.md`, затем пройти сценарий «голос → отчёт → правка → исключение → восстановление».

Отсутствие дубля в одном ручном запуске не заменяет архитектурную идемпотентность, которая пока не реализована.
