## 1.0 — Curator-ready MVP

Цель: куратор воспроизводит учебный сценарий по документации без догадок.

- локальные `lint`, `test`, `build` проходят;
- все миграции применены к демонстрационному Supabase;
- Vercel deployment содержит требуемые переменные;
- Telegram webhook установлен с секретом и без `last_error_message`;
- голос создаёт запись, продажу и позиции;
- Web App показывает отчёт, правку, исключение и восстановление;
- ограничения демонстрационного режима явно указаны.

Критерий: пункты единственного активного [curator release plan](../plans/active/003-curator-release/acceptance.md) подтверждены фактами.

## 1.1 — Production hardening

До публичного тестирования:

- гарантировать идемпотентность Telegram update;
- сохранять продажу и позиции транзакционно;
- определить политику хранения аудио и данных;
- добавить интеграционные и браузерные тесты;
- устранить скрытый fallback ошибок чтения в пустые/демонстрационные данные.

Критерий: повтор или отказ не создаёт дубль и не оставляет частичное состояние.

## 1.2 — Owner auth and shop isolation

До реальных коммерческих данных:

- проверять Telegram init data или эквивалентную серверную сессию;
- связать владельца с магазином;
- выводить `shop_id` из сессии;
- ограничить все чтения, server actions, RPC и Storage;
- заменить demo RLS на deny-by-default политики;
- добавить негативные тесты доступа.

Критерий: магазин A не может прочитать или изменить данные магазина B.

## 1.3 — Monitoring, backups, GitHub Actions

До поддерживаемой эксплуатации:

- ввести correlation ID, метрики и alerting для webhook/STT/LLM/Supabase;
- настроить backup БД и Storage и регулярно проверять restore;
- добавить GitHub Actions для lint, test, build и защитных проверок;
- документировать runbook, rollback, RPO и RTO.

Критерий: отказ обнаруживается автоматически, восстановление проверено, а ошибочный PR блокируется CI.

## Правило активации

Одновременно активен только ближайший этап. Следующие версии остаются в `docs/plans/backlog` и не реализуются без явного переноса в `docs/plans/active`.
