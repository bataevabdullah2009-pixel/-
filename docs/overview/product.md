# Продукт

Магазину нужен быстрый способ фиксировать продажи без отдельного рабочего места. Продавец произносит товар, количество и цену, а владелец получает структурированный журнал и сводную выручку.

Поток данных:

`Telegram voice → download → STT → LLM parser → Supabase → owner report`

Результат считается активной продажей только когда позиция имеет статус `processed` и `deleted_at is null`. Неоднозначные позиции остаются в журнале и направляются на ручное исправление.
