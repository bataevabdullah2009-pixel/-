# Продукт

Магазину нужен быстрый способ фиксировать продажи без отдельного рабочего места. Продавец произносит товар, количество и цену, а владелец получает структурированный журнал и сводную выручку.

Поток данных:

`Telegram voice → download → STT → LLM parser → Supabase → owner report`

Каждая распознанная запись сначала направляется в «Нужно проверить». Пользователь сохраняет исправления отдельно от подтверждения. Результат считается активной продажей только после подтверждения, когда позиция имеет внутренний статус `processed` и `deleted_at is null`; в UI этот статус называется «Подтверждено».
