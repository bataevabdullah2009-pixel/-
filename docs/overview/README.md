# Обзор

«Голосовой журнал продаж» помогает небольшому магазину вести продажи голосом. Продавец отправляет voice-сообщение в Telegram, система распознаёт товар, количество и цену, сохраняет продажу и показывает владельцу мобильный отчёт.

## Основной сценарий

1. Продавец диктует продажу.
2. Bot скачивает voice.
3. STT возвращает текст.
4. Parser выделяет товары.
5. Supabase сохраняет `voice_records`, `sales`, `sale_items`.
6. Уверенная запись сразу становится `processed`.
7. Сомнительная запись становится `needs_review`.
8. Сомнительная запись подтверждается или отменяется в Telegram либо во вкладке WebApp «Проверка» после server-side проверки магазина.
9. Владелец смотрит отчёт, записи, проверку и продавцов в WebApp.

## WebApp modes

Mini App открывается двумя штатными способами:

- Telegram mode — через Telegram `web_app` кнопку с валидным raw `initData`.
- Browser fallback mode — напрямую в браузере, если `ALLOW_WEBAPP_FALLBACK=true` и сервер знает `DEFAULT_SHOP_ID` / `DEFAULT_SELLER_ID`.

Клиент не передаёт доверенный `shop_id`.

## Product state

Уверенные позиции сразу попадают в отчёт.

Review-записи не входят в выручку до явного confirm.

Telegram `❌ Отмена` или WebApp cancel во вкладке «Проверка» переводит запись в `cancelled` и soft-delete active items.

WebApp показывает отдельную вкладку «Проверка» для сомнительных записей: parsed text, товары, edit/delete и confirm/cancel.

Владелец редактирует товары компактно через карандаш.

Корзина выполняет soft delete с подтверждением.

Auth/DB ошибки не маскируются нулевым отчётом.

Diagnostics доступны в production только при `DEBUG_TELEGRAM_WEBAPP=true`.

## Границы MVP

Входит:

- голосовая фиксация продаж;
- Telegram confirm/cancel и WebApp «Проверка» для сомнительных записей;
- отчёт;
- журнал записей;
- проверка сомнительных записей;
- продавцы;
- ручная корректировка товара;
- soft delete и restore.

Не входит:

- CRM;
- склад;
- касса;
- оплаты;
- клиентская база.
