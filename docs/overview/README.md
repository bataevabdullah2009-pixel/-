# Обзор продукта

`Голосовой журнал продаж` помогает магазину вести выручку через голосовые сообщения Telegram. Продавец не заполняет таблицу вручную: он говорит продажу, бот распознаёт товары и цены, Supabase хранит запись, WebApp показывает отчёт.

## Основной поток

1. Продавец открывает Telegram-бота.
2. `/start` проверяет привязку к магазину и даёт доступ к WebApp.
3. Продавец отправляет голосовое сообщение.
4. Бот скачивает аудиофайл.
5. Аудио подготавливается для STT.
6. STT возвращает русскую расшифровку.
7. Парсер/LLM выделяет товары, количество, единицы, цены, суммы и уверенность.
8. Supabase сохраняет `voice_records`, `sales`, `sale_items`.
9. Бот отвечает пользователю.
10. WebApp показывает обновлённые данные.

## Статусы

- `processed` - запись подтверждена и входит в выручку.
- `needs_review` - запись сохранена, но не входит в выручку.
- `cancelled` - пользователь отменил запись, она не входит в выручку.
- `failed` - голосовой конвейер не смог обработать запись.

## Уверенная запись

Уверенная запись имеет валидные товары, количество, цену, сумму и достаточную уверенность.

Результат:

- `sales.status = processed`;
- `voice_records.status = processed`;
- валидные `sale_items.status = processed`;
- `sales.total_amount` равен сумме активных обработанных позиций;
- бот отвечает `✅ Запись сохранена: ...`;
- запись сразу видна в отчёте.

## Сомнительная запись

Сомнительная запись появляется, если не хватает товара, количества, цены, суммы, уверенность низкая или сработал резервный парсер.

Результат:

- `sales.status = needs_review`;
- `voice_records.status = needs_review`;
- активные позиции на проверке не входят в выручку; активные позиции `processed` могут учитываться;
- бот отвечает предупреждением;
- под сообщением только `✅ Подтвердить` и `❌ Отмена`;
- WebApp показывает запись на вкладке `Проверка`.

## Подтверждение/отмена

`✅ Подтвердить`:

- переводит продажу в `processed`;
- переводит голосовую запись в `processed`;
- переводит все валидные активные позиции в `processed`;
- оставляет неполные активные позиции смешанной корзины в `needs_review`;
- пересчитывает `sales.total_amount`;
- добавляет товары в отчёт.

`❌ Отмена`:

- переводит продажу в `cancelled`;
- переводит голосовую запись в `cancelled`;
- мягко удаляет активные позиции;
- сохраняет данные для аудита;
- не добавляет выручку.

## WebApp

WebApp содержит:

- `Отчёт`;
- `Проверка`;
- `Записи`;
- `Продавцы`.

WebApp использует спокойный премиальный SaaS-стиль: фон `#0B1020`, поверхности `#12192B`/`#161F34` и синий акцент `#5B8CFF`; предупреждающий оранжевый применяется только для состояний проверки.

Сомнительные голосовые записи подтверждаются или отменяются inline-кнопками Telegram под исходным сообщением либо на вкладке WebApp `Проверка`. Оба пути используют одинаковые правила статусов и выручки.

## Выручка

Выручка рассчитывается по активным обработанным позициям только из обработанных продаж. Это защищает отчёт от частично распознанных или отменённых голосовых записей.
