# Структура каталогов

```text
voice-sales-log/
├── apps/
│   ├── web/       # Next.js, Telegram Web App, серверные действия и webhook
│   └── bot/       # обработчики Telegram, сервисы и локальный polling
├── packages/
│   └── shared/    # типы, Zod-схемы, парсер, даты и расчёт отчёта
├── supabase/
│   ├── migrations/# версионируемые изменения схемы SQL
│   └── seed.sql   # демонстрационный магазин, продавец и товары
├── docs/
│   ├── overview/  # точка входа в документацию
│   ├── specs/     # глобальные, технические, продуктовые и информационные контракты
│   ├── features/  # каталог функций
│   ├── plans/     # активные, завершённые и будущие задачи
│   ├── rules/     # инженерные, продуктовые и безопасностные ограничения
│   ├── stories/   # пользовательские истории по ролям
│   ├── architecture/
│   └── roadmap/
├── scripts/       # установка и диагностика webhook
├── tests/         # модульные и регрессионные тесты Vitest
└── codex/skills/voice-sales-log/ # основной проектный навык
```

## Ответственность

- `apps/web/src/app` содержит сегменты страниц; `/daily-report` владеет мутациями отчёта, `/api/telegram/webhook` является единственным внешним HTTP API.
- `apps/web/src/features/records` содержит серверные запросы Supabase и оркестрацию мутаций.
- `apps/bot/src/core/process-update.ts` — общая точка обработки webhook и polling.
- `apps/bot/src/handlers` отвечает за пользовательское поведение Telegram.
- `apps/bot/src/services` интегрирует аудио, STT, LLM, Storage и БД.
- `packages/shared` содержит детерминированную логику без внешних сервисов.
- `supabase/migrations` содержит каждое изменение схемы; ручная несинхронизированная правка удалённой БД запрещена.
- документация размещается в профильном каталоге; новые файлы `README.md` и случайные файлы в корне `docs` запрещены.
