# Расчёт отчёта

## Источник и группировка

Отчёт строится из `sale_items`, связанных с продажами выбранного диапазона. В количество и выручку входят только позиции с `deleted_at is null`, `status = processed` и ненулевыми полями `price` и `total`. `confidence` используется при первоначальном назначении статуса, но не является отдельным фильтром уже обработанной позиции.

Группировка выполняется по `product_id`, иначе по нормализованному `product_name`. Варианты единиц приводятся к `шт` или `кг`.

## Периоды

Календарные границы явно рассчитываются в `Europe/Moscow`, поэтому UTC-среда Vercel не переносит позднюю продажу в другой день владельца.

- день: сегодня, вчера или выбранная дата `[00:00, следующие 00:00)`;
- неделя: последние 7 календарных дней;
- месяц: выбранный календарный месяц;
- год: выбранный календарный год.

## Итоги

- итог позиции: `quantity × price`, округление до 2 знаков;
- выручка строки: сумма итогов позиций;
- общая выручка: сумма строк отчёта;
- общее количество: сумма количества строк.

Текущий MVP ошибочно объединяет разные единицы в одном общем количестве; строки сохраняют единицу. `needs_price`, `needs_review`, `failed`, `excluded` и все строки с `deleted_at is not null` не входят в выручку. После мутации `sales.total_amount` пересчитывается по тем же правилам. `getReport` возвращает итог, активные и исключённые карточки; `getReviewItems` возвращает активные позиции проверки.
