# LLM-парсер

## Пример

Вход: `Сникерс 5 штук по 60 рублей`

```json
{
  "items": [
    {
      "product_name": "Сникерс",
      "quantity": 5,
      "unit": "шт",
      "price": 60,
      "total": 300,
      "confidence": 0.95
    }
  ],
  "raw_text": "Сникерс 5 штук по 60 рублей",
  "cleaned_text": "Сникерс — 5 штук по 60 рублей.",
  "needs_review": false
}
```

Статус вычисляется приложением после парсера и подстановки цены товара, а не принимается от модели.

## Правила

- не придумывать товар, цену и количество;
- сохранять все товары и исходный порядок;
- нет цены и цены товара по умолчанию → `needs_price`, итог `null`;
- уверенность ниже `0.75`, пустое имя или отсутствующее количество → `needs_review`;
- приложение пересчитывает `total = quantity × price` независимо от ответа LLM;
- `рублей`, `руб`, `₽` и `по <число>` являются допустимыми признаками цены;
- `штук` и `шт` являются признаками количества в штуках, `кг` и `килограмм` — веса;
- число без явного признака количества или цены не угадывается;
- допустимая ручная правка задаёт уверенность `1` и `processed`.

LLM вызывается с температурой `0` и `response_format: json_object`; ответ проверяется Zod и `enforceTranscriptEvidence`.
