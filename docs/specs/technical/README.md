## Инварианты

- рабочий Telegram update входит только через `/api/telegram/webhook`;
- polling используется только локально;
- парсер не придумывает товар, цену или количество;
- схема меняется только новой миграцией;
- service role остаётся в серверном коде;
- исключение товара является мягким удалением;
- отчёт строится только из активных допустимых позиций.
