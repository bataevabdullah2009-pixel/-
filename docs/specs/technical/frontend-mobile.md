# Мобильный интерфейс

## Страницы

- `/` перенаправляет на `/daily-report`.
- `/daily-report` показывает сводку, фильтр дат, отчёт, управление позициями и исключённые позиции.
- `/records` показывает карточки продаж, ссылку на аудио и фильтры даты, продавца и текста.
- `/sellers` показывает список продавцов.

## Контракт мобильного интерфейса

- целевая среда — узкий экран Telegram Web App с сенсорными элементами и нижним безопасным отступом;
- на телефоне отчёт использует карточки, на компьютере — таблицу;
- закреплённая сводка показывает период, выручку, количество, обновление и сброс дня;
- нижняя навигация использует векторные значки и активное состояние;
- загрузка использует каркас, пустое состояние объясняет следующее действие;
- на ширине 390 px нет горизонтальной прокрутки;
- даты и ссылки используют тот же календарь `Europe/Moscow`, что и запросы отчёта.

## Управление позициями

Каждая активная карточка позволяет изменить имя, положительное количество и положительную цену. Сохранение обновляет позицию по ID, пересчитывает итог и продажу, ставит `processed` и убирает исправленную строку из блока проверки. Исключение требует подтверждения и выполняет soft delete со статусом `excluded`. Сброс доступен только для однодневного периода. Панель исключённых позиций объясняет хранение и позволяет восстановление.

Опасные действия визуально отделены, сообщение об успехе или безопасной ошибке показывается после перенаправления. Технические ответы Supabase не выводятся. Интерфейс не редактирует исходную расшифровку или JSON парсера.

Состояние ошибки чтения данных сейчас не реализовано полностью: ошибки Supabase часто отображаются как пустой список, а отсутствие конфигурации включает встроенные демонстрационные данные. Интерфейс должен различать пустой результат, демонстрационный режим и отказ сервиса.
