# Обработка голоса

## Успешный путь

1. Telegram доставляет голосовое сообщение.
2. Обработчик получает ссылку и скачивает OGG/Opus.
3. Исходное аудио загружается в приватный Supabase Storage; для STT готовится MP3 или резервный OGG.
4. STT-запрос отправляет multipart-поля файла, модели и формата ответа.
5. Исходная расшифровка записывается в аудит.
6. LLM очищает текст и возвращает строгий JSON; приложение повторно проверяет числовые признаки.
7. `voice_records` сохраняет путь аудио, тексты, JSON парсера, статус и ошибку.
8. `sales` сохраняет текст, сумму и статус.
9. `sale_items` сохраняет каждую позицию в исходном порядке.
10. Бот отвечает очищенным текстом и статусом.

## Ошибки

- Ошибка скачивания переводит обработку в общий `catch`; при возможности сохраняется голосовая запись со статусом `failed`.
- Ошибка STT журналирует статус и метаданные файла после маскирования.
- Ошибка очистки LLM использует локальное исправление регистра и пунктуации.
- HTTP-ошибка парсера создаёт продажу `needs_review` без позиций и сохраняет описание ошибки.
- Недопустимый или пустой JSON приводит к исключению и сохранению доступных диагностических данных.
- Ошибка Supabase может оставить частично созданные строки, потому что общей транзакции нет.
- Повторная доставка Telegram может создать дубликат, потому что идемпотентность не реализована.

Ключи и bearer-токены не должны попадать в аудит или ответ продавцу.
