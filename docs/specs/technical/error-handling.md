# Обработка ошибок

Сервер логирует контекст операции и исходную ошибку. Пользователь получает стабильные сообщения без SQL, stack trace, ключей и внутренней структуры:

- «Не удалось загрузить отчёт.»
- «Не удалось сохранить товар.»
- «Не удалось исключить товар из отчёта.»
- «Откройте Web App через Telegram.»

Ошибка доступа продавца обрабатывается отдельно и не создаёт `voice_records`, `sales` или `sale_items`.
