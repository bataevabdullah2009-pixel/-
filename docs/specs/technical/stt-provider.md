# Провайдер STT

STT получает аудио и возвращает буквальную расшифровку. Он не структурирует продажу и не придумывает данные.

## Контракт

- Адрес: `STT_API_URL`, Whisper-совместимый API расшифровки.
- Авторизация: `Authorization: Bearer STT_API_KEY`.
- Модель: `STT_MODEL`, по умолчанию `whisper-large-v3-turbo`.
- Multipart-поля: `file`, `model`, `response_format=json`.
- Предпочтительный файл: `voice.mp3`, `audio/mpeg`.
- Резервный вариант: исходный Telegram OGG при недоступном или ошибочном ffmpeg.

Точное имя модели зависит от провайдера и задаётся окружением.

## Поведение при ошибке

При ответе не 2xx сервис бросает исключение со статусом и техническими метаданными аудио. Обработчик маскирует чувствительный текст, пытается сохранить запись `failed` и просит продавца повторить. Пустая расшифровка не может считаться `processed`.
