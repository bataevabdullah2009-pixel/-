# Backup и restore

## Что резервировать

- Supabase Postgres: схема, миграции и данные бизнес-таблиц;
- Supabase Storage: исходное аудио и метаданные объектов;
- конфигурацию bucket/policies как код, но не значения секретов;
- историю миграций и версию приложения, совместимую с backup.

Резервная копия БД без Storage не восстанавливает голосовой след, а копия Storage без БД теряет связи. Политика должна определить RPO, RTO, частоту, срок хранения, шифрование и ответственного.

## Восстановление

1. Выбрать чистое изолированное окружение.
2. Восстановить Postgres совместимым способом.
3. Восстановить объекты Storage с теми же путями либо выполнить документированное переназначение.
4. Проверить внешние ключи, counts, подписанные ссылки и отчёт.
5. Выполнить smoke-сценарий без переключения production-трафика.
6. Зафиксировать фактические RPO/RTO и найденные расхождения.

## Критерии приёмки

- автоматический backup покрывает БД и аудио;
- доступ к копиям ограничен и проверяется;
- тестовый restore выполняется по инструкции другим участником;
- восстановленные продажи, позиции, аудит и аудио согласованы;
- достигнуты утверждённые RPO/RTO;
- дата последней успешной проверки и результат зафиксированы.
