# Журнал аудита

| Действие | Когда | Ключевые данные |
| --- | --- | --- |
| `stt_raw_text_received` | STT успешно ответил | ID сообщения, исходный текст |
| `llm_parser_json_received` | Получен ответ парсера | JSON парсера или ошибка |
| `sale_items_created` | Позиции добавлены | ID продажи и записи, количество, статусы |
| `sale.processed` | Продажа сохранена | Статус и число позиций |
| `voice.failed` | Обработка завершилась ошибкой | ID записи, замаскированная ошибка |
| `sale_item_updated` | Владелец сохранил правку | ID позиции, состояние до и после |
| `sale_item_deleted` | Ручное исключение | Позиция, имя, время и причина |
| `sale_item_restored` | Восстановление | Предыдущие метаданные удаления |
| `daily_revenue_reset` | Сброс одного дня | Диапазон, время, число позиций и ID продаж |

Интерфейс не редактирует строки аудита и не использует их напрямую в расчёте выручки. В текущем MVP в аудите нет идентификатора владельца из-за отсутствия его авторизации; после её добавления каждое административное событие должно хранить действующего пользователя.
