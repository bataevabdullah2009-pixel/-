# Reports Spec

## Назначение отчётов

Отчёты нужны владельцу магазина, чтобы быстро увидеть итог продаж без чтения длинных голосовых записей.

## Страница

URL: `/daily-report`

Заголовок:

```text
Отчёт за день
```

## Фильтры

- Сегодня.
- Вчера.
- Неделя.
- Месяц.
- Год.
- Выбор даты вручную.

## Основная таблица

| Колонка | Что показывает |
| --- | --- |
| Товар | Нормализованное название товара. |
| Количество | Сумма количества по одинаковому товару. |
| Выручка | Сумма `total` только processed-позиций. |

## Группировка

Одинаковые товары группируются так:

1. По `product_id`, если он есть.
2. По нормализованному названию товара, если `product_id` нет.

Если часть позиций уже связана с `product_id`, а часть имеет только такое же нормализованное название,
позиции без `product_id` присоединяются к существующей product-группе.

Название товара перед группировкой:

- обрезается по краям;
- приводится к нижнему регистру для поиска;
- сопоставляет простые формы: `хлеб`, `Хлеб`, `хлеба` -> `Хлеб`; `молоко`, `молока` -> `Молоко`.

Единицы `штука`, `штуки`, `штук`, `шт.` приводятся к `шт`.
Если unit не указан, используется `шт`.

Пример:

| Голосовая запись | Позиция |
| --- | --- |
| `хлеб 3 штуки по 40` | Хлеб, 3 шт, 120 ₽ |
| `Хлеб 2 шт по 40` | Хлеб, 2 шт, 80 ₽ |

В отчёте:

| Товар | Количество | Выручка |
| --- | ---: | ---: |
| Хлеб | 5 шт | 200 ₽ |

## Нужно проверить

В блок попадают позиции:

- `needs_price`;
- `needs_review`;
- `failed`;
- processed-позиции с `price = null`;
- позиции с confidence ниже 0.75.

Форма ручного исправления позволяет изменить:

- `product_name`;
- `quantity`;
- `price`.

После сохранения:

- `total` пересчитывается как `quantity * price`;
- `sale_items.status` становится `processed`;
- связанная продажа пересчитывает `total_amount`;
- запись появляется в основной таблице отчёта, если все обязательные поля валидны.

## Запреты отчёта

Основная таблица не показывает:

- продавца;
- время каждой продажи;
- raw transcript;
- status;
- id;
- технические ошибки.
