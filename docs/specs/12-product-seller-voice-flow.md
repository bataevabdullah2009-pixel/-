# Голосовой сценарий продавца

Статус: реализовано.

## Цель

Продавец должен быстро сохранить продажу голосом и получить понятный ответ.

Если система уверена, продажа сразу входит в отчёт.

Если система сомневается, продавец принимает решение в Telegram или на WebApp вкладке `Проверка`.

## Уверенный пример

```text
Сникерс, 3 штуки по 200 рублей. Буханка хлеба, 5 штук по 50 рублей.
```

Ожидаемый результат:

1. `sale_items[0]`: `Сникерс`, quantity `3`, unit `шт`, price `200`, total `600`.
2. `sale_items[1]`: `Буханка хлеба`, quantity `5`, unit `шт`, price `50`, total `250`.
3. Sale total: `850`.
4. Confidence `>= 0.80`.
5. Статус item `processed`.
6. Статус sale `processed`.
7. Статус voice record `processed`.
8. Bot отвечает:

```text
✅ Запись сохранена: Сникерс, 3 штуки по 200 рублей. Буханка хлеба, 5 штук по 50 рублей.
```

Запись сразу входит в отчёт.

## Сомнительный пример

```text
Буханка хлеба три по двадцать, масло двести грамм сто, шашлык десять килограмм пятьсот
```

Если parser считает запись неоднозначной, результат:

1. Sale status `needs_review`.
2. Voice record status `needs_review`.
3. Items не входят в выручку.
4. Bot отвечает:

```text
⚠️ Запись сохранена, но нужно подтвердить товары и цены.
Распознано: ...
```

Под сообщением есть:

```text
✅ Подтвердить
❌ Отмена
```

## Подтверждение

Нажатие `✅ Подтвердить` в Telegram или `Подтвердить` в WebApp:

1. Переводит sale в `processed`.
2. Переводит voice record в `processed`.
3. Делает валидные items `processed`.
4. Неполные mixed-cart items оставляет `needs_review`.
5. Добавляет выручку в отчёт.
6. Bot пишет:

```text
✅ Подтверждено: N позиций, сумма X ₽
```

## Отмена

Нажатие `❌ Отмена` в Telegram или `Отмена` в WebApp:

1. Переводит sale в `cancelled`.
2. Переводит voice record в `cancelled`.
3. Выполняет soft-delete active items.
4. Не добавляет выручку.
5. Bot пишет:

```text
❌ Запись отменена и не входит в отчёт.
```

## Ошибки

Сообщение об успехе отправляется только после read-back проверки sale и sale_items.

При ошибке Supabase bot отвечает:

```text
⚠️ Не удалось сохранить запись. Попробуйте ещё раз.
```

При неизвестном seller:

```text
Ваш Telegram не привязан к магазину.
```

## Вне области

1. Выбор магазина клиентом.
2. CRM/склад/касса.
3. Изменение STT/LLM без отдельной задачи.
