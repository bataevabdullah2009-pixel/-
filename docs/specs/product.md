# Product Spec

## Цель продукта

Сделать простой голосовой журнал продаж для небольшого магазина.

Продавец говорит продажу голосом, владелец видит текстовую запись и итоговую таблицу по товарам.

## Пользователи

| Пользователь | Что делает |
| --- | --- |
| Продавец | Отправляет голосовое сообщение в Telegram-бот. |
| Владелец | Смотрит записи, отчёты и спорные позиции. |
| Преподаватель | Проверяет запуск, архитектуру, docs и тесты. |

## Основной пользовательский сценарий

1. Продавец говорит: `хлеб 3 по 40 молоко 2 по 90`.
2. Бот отвечает: `Голосовое получено, обрабатываю.`
3. Система распознаёт текст.
4. Система очищает текст.
5. Система извлекает позиции.
6. Система сохраняет продажу.
7. Бот отвечает продавцу, что запись сохранена.
8. Владелец открывает `/daily-report`.
9. Владелец видит таблицу:
   - Хлеб, 3 шт, 120 ₽.
   - Молоко, 2 шт, 180 ₽.
   - Итого, 5, 300 ₽.

## MVP

MVP должен доказать, что голосовой ввод можно превратить в полезный дневной отчёт.

Обязательные возможности:

- голосовой ввод через Telegram;
- сохранение аудио;
- сохранение текста;
- сохранение структурированных позиций;
- отчёт по товарам;
- блок проверки;
- ручное исправление цены.

## Не цель проекта

Проект не решает задачи:

- складского учёта;
- кассового учёта;
- фискализации;
- оплаты;
- CRM;
- сложной BI-аналитики.

## Product Success Criteria

Проект успешен, если преподаватель может:

- открыть README и запустить проект;
- прочитать global spec и понять границы;
- увидеть таблицы Supabase;
- увидеть понятный Telegram flow;
- увидеть веб-отчёт;
- запустить тесты.
