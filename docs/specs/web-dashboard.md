# Web Dashboard Spec

## Назначение

Веб-панель нужна владельцу магазина для просмотра записей, продавцов и отчёта.

## Страницы

| URL | Назначение |
| --- | --- |
| `/` | Redirect на `/daily-report`. |
| `/daily-report` | Таблица итогов продаж. |
| `/records` | Список голосовых записей и продаж. |
| `/sellers` | Список продавцов. |

## `/records`

Показывает:

- дату;
- время;
- продавца;
- очищенный текст;
- статус;
- выручку по продаже;
- ссылку “Прослушать аудио”, если есть signed URL.

Фильтры:

- сегодня;
- неделя;
- месяц;
- год;
- ручная дата;
- продавец;
- поиск по тексту.

## `/daily-report`

Показывает:

- заголовок;
- фильтры периода;
- таблицу “Товар / Количество / Выручка”;
- строку “Итого”;
- блок “Нужно проверить”;
- форму ручного исправления позиции.

Форма ручного исправления позиции:

- показывает спорные позиции `needs_price` и `needs_review`;
- позволяет изменить товар;
- позволяет изменить количество;
- позволяет изменить цену;
- после сохранения пересчитывает total;
- переводит исправленную позицию в `processed`;
- возвращает позицию в основной отчёт, если товар, количество и цена валидны.

## UI rules

- Интерфейс должен быть спокойным и рабочим.
- Не использовать маркетинговую landing-страницу.
- Не перегружать отчёт техническими данными.
- Карточки использовать только для записей и блоков формы.
- Текст должен помещаться на мобильном экране.
