# Web Dashboard Spec

## Назначение

Веб-панель нужна владельцу магазина для просмотра записей, продавцов и отчёта.

## Страницы

| URL | Назначение |
| --- | --- |
| `/` | Redirect на `/daily-report`. |
| `/daily-report` | Таблица итогов продаж. |
| `/records` | Список голосовых записей и продаж. |
| `/sellers` | Список продавцов. |

## `/records`

Показывает:

- дату;
- время;
- продавца;
- очищенный текст;
- статус;
- выручку по продаже;
- ссылку “Прослушать аудио”, если есть signed URL.

Фильтры:

- сегодня;
- неделя;
- месяц;
- год;
- ручная дата;
- продавец;
- поиск по тексту.

## `/daily-report`

Показывает:

- заголовок;
- фильтры периода;
- таблицу “Товар / Количество / Выручка”;
- строку “Итого”;
- блок “Нужно проверить”;
- форму ручного исправления позиции.

Форма ручного исправления позиции:

- показывает спорные позиции `needs_price` и `needs_review`;
- позволяет изменить товар;
- позволяет изменить количество;
- позволяет изменить цену;
- после сохранения пересчитывает total;
- переводит исправленную позицию в `processed`;
- возвращает позицию в основной отчёт, если товар, количество и цена валидны.

## UI rules

- Интерфейс должен быть спокойным и рабочим.
- Не использовать маркетинговую landing-страницу.
- Не перегружать отчёт техническими данными.
- Текст должен помещаться на мобильном экране.
- На desktop отчёт показывает таблицу, а на mobile — карточки товара с количеством, единицей и выручкой.
- На mobile не должно быть горизонтального скролла.
- Итоги периода закреплены сверху и показывают период, общую выручку и общее количество.
- Фильтры периода компактно переносятся на узком экране.
- Кнопка “Обновить” повторно загружает Server Component и актуальные данные Supabase.
- На mobile используется нижняя навигация “Отчёт / Записи / Продавцы”; на desktop остаётся верхняя.
- Записи на телефоне показывают статус, очищенный текст, сумму и доступные действия для аудио и исправления.
- Во время переходов показывается loading state, а при отсутствии данных — понятный empty state.
