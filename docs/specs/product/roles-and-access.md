# Роли и доступ

## Owner

Владелец открывает Web App кнопкой типа `web_app`. Сервер валидирует подпись и срок `initData`, находит активную запись `owners` по `telegram_id` либо существующую активную seller-привязку периода rollout, получает `shop_id` из базы и проверяет существование магазина. Владелец видит и изменяет только данные этого магазина.

## Seller

Продавец взаимодействует с ботом. Для сохранения и совместимого доступа к отчёту требуется активная запись `sellers` с совпадающим `telegram_id` и существующим магазином. Неизвестному или неактивному пользователю доступ запрещён.

## System

Система скачивает голосовое, выполняет STT и LLM parsing, нормализует позиции, сохраняет продажу и строит отчёты. Service role используется только в серверных модулях.
