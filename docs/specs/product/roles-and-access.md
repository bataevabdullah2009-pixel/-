# Роли и доступ

## Owner

Владелец открывает Web App из Telegram. Сервер валидирует подпись и срок `initData`, находит активную запись `owners` по `telegram_id` и получает `shop_id` из базы. Владелец видит и изменяет только данные своего магазина.

## Seller

Продавец взаимодействует с ботом. Для сохранения требуется активная запись `sellers` с совпадающим `telegram_id` и назначенным магазином. Неизвестному пользователю бот отвечает: «Ваш Telegram не привязан к магазину. Обратитесь к владельцу.»

## System

Система скачивает голосовое, выполняет STT и LLM parsing, нормализует позиции, сохраняет продажу и строит отчёты. Service role используется только в серверных модулях.
