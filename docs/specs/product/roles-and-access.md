# Роли и доступ

## Owner

Владелец открывает Web App кнопкой типа `web_app`. Сервер валидирует подпись и срок `initData`, находит активную запись `owners` по `telegram_id` либо существующую активную seller-привязку периода rollout, получает `shop_id` из базы и проверяет существование магазина. Владелец видит, исправляет, подтверждает и исключает только данные этого магазина.

## Seller

Продавец взаимодействует с ботом и может открыть доступный ему Mini App для проверки записи. Для сохранения и совместимого доступа к отчёту требуется активная запись `sellers` с совпадающим `telegram_id` и существующим магазином. Неизвестный пользователь получает `SELLER_NOT_LINKED`, неактивный — `SELLER_INACTIVE`.

## System

Система скачивает голосовое, выполняет STT и LLM parsing, нормализует позиции, сохраняет продажу и строит отчёты. Service role используется только в серверных модулях.
