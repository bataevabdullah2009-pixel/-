# Голосовой поток продавца

1. Бот получает Telegram user id и вызывает `requireSeller()`.
2. Для активного продавца скачивается OGG-файл; сохранение в Storage выполняется best-effort.
3. Аудио подготавливается для STT.
4. Расшифровка очищается и передаётся LLM parser.
5. Числа сверяются с исходным текстом.
6. Позиции нормализуются и сохраняются функцией `save_voice_sale`; до rollout RPC доступен совместимый server-side insert.
7. Бот сообщает итоговый статус.

В production неизвестные пользователи не создаются автоматически. При `DEMO_MODE=true` разрешено создать продавца в явно настроенном демонстрационном магазине.
