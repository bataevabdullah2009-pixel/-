# Панель владельца

Панель содержит сводку подтверждённого количества и выручки, блок «Нужно проверить», подтверждённые продажи, исключённые позиции, журнал записей и список продавцов.

Вход в панель начинается только из Telegram `web_app`: reply, inline и menu buttons используют один production URL. Корневая страница не выполняет redirect до Telegram bootstrap. До успешной проверки initData защищённые данные не показываются; UI различает отсутствующий/невалидный initData, непривязанный Telegram, inactive binding и отсутствующий магазин. `/debug-telegram` показывает только безопасную диагностику.

Все чтения и изменения выполняются на сервере после `requireOwner()`. Фильтр магазина добавляется сервером. Сохранение правок оставляет позицию в проверке; подтверждение переводит её в «Подтверждено». После изменения маршруты `/daily-report` и `/records` инвалидируются без полной перезагрузки документа.
