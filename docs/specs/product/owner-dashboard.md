# Owner Dashboard

Владелец видит:

- отчёт по выручке и количеству;
- блок «Нужно проверить»;
- список готовых товаров;
- исключённые товары с восстановлением;
- журнал записей;
- список продавцов.

Все чтения и изменения выполняются на сервере после `resolveRequestContext()` / `requireOwner()`. Seller shop, найденный по Telegram user id, совпадает с bot save. Report получает sale IDs только из этого shop и затем читает связанные items. Auth/DB error показывается явно без нулевых метрик и ложного empty state.

Сохранение валидных товара, количества и цены переводит позицию в «Готово» и сразу пересчитывает отчёт. Карточка раскрывается только по карандашу, показывает pending/error state и не очищает ввод при ошибке. Корзина выполняет soft delete после локального подтверждения. Исключение и восстановление проверяют изменённую строку и обновляют UI через revalidation без ручной перезагрузки.

Подробности: [`webapp-report.md`](./webapp-report.md) и [`sale-item-editing.md`](./sale-item-editing.md).
