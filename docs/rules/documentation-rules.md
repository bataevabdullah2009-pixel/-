# Правила документации

- `README.md` — вход для человека, `AGENTS.md` — вход для агента, `docs/INDEX.md` — глобальная карта;
- тематический `README.md` индексирует только свой каталог и не дублирует глобальную карту;
- спецификация описывает контракт, план — работу, roadmap — порядок версий, CHANGELOG — сделанные изменения;
- планируемое нельзя называть реализованным;
- после переименования обновить все ссылки и удалить устаревший документ;
- каждый каталог `docs` содержит понятный Markdown-файл;
- не создавать противоречащие дубли ради нового названия;
- проектный текст писать на русском, сохраняя точные технические идентификаторы;
- внешнюю проверку фиксировать с датой и фактом, а не выводить из локального теста;
- если задача не входит в MVP, записать её в backlog, не добавляя сразу в код.
