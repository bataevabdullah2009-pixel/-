## Непереговорные правила

- Проект остаётся простым голосовым журналом продаж.
- Нельзя превращать проект в CRM, склад, кассу или товароучёт.
- ИИ не имеет права выдумывать товары, цены, категории и аналитику.
- Если цена неизвестна, позиция получает `needs_price`.
- Позиции без цены не попадают в выручку.
- Основной отчёт показывает только товар, количество и выручку.
- Все ошибки логируются.
- Секреты не хранятся в коде.
- Telegram webhook проверяет `x-telegram-bot-api-secret-token`.
- Polling не запускается на Vercel.
- После изменений обновляются `CHANGELOG.md`, планы и нужные specs.

## Правило изменения проекта

Любое изменение проходит через 4 вопроса:

1. Это помогает главному сценарию “голосовое -> текст -> продажа -> отчёт”?
2. Это не превращает проект в CRM, склад или кассу?
3. Это описано в specs/features/plans?
4. Это проверено lint/test/build?

Если хотя бы один ответ “нет”, изменение нельзя считать готовым.
