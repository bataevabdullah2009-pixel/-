# Правила данных

- `shop_id` не принимается от клиента.
- Service role key используется только сервером.
- Активный отчёт учитывает только `processed` и `deleted_at is null`.
- Выручка требует валидный `total`; цена за единицу может быть сохранена или выведена из total.
- Подтверждение работает по активным `sale_items`: валидные позиции входят в выручку, неполные остаются `needs_review`.
- Резервный парсер должен создавать отдельные `sale_items` для каждого товара и отдельные строки проверки для неполных остатков.
- Валидное редактирование WebApp ставит позицию `processed`; такая позиция может войти в выручку, даже если родительская продажа остаётся `needs_review`.
- Исключение позиции выполняется только через soft delete.
- Голосовая запись на проверке может содержать обработанные позиции выручки; неполные позиции проверки не входят в выручку.
- Отмена voice-записи хранится как `cancelled` и не удаляет sale физически.
- Изменение схемы требует миграции и обновления документации данных.
- Устаревший `needs_price` поддерживается в чтении, но новые неполные позиции используют `needs_review`.
