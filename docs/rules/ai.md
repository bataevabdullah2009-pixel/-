# Правила AI

LLM извлекает только факты из STT-текста. Количество и цена подтверждаются детерминированным слоем доказательств. Нельзя выдумывать цену, количество или товар.

`confidence >= 0.80` требуется для автоматического `processed`. Низкая уверенность, отсутствующая цена/количество, странный текст или резервный парсер дают `needs_review`.

Резервный парсер должен исправлять вывод LLM, который вернул один длинный текст вместо массива товаров: валидные товары разделяются по доказательствам, а неполные фрагменты сохраняются отдельными позициями на проверку.

`needs_review` не означает потерю продажи: бот сохраняет запись и показывает кнопки Telegram `✅ Подтвердить` и `❌ Отмена`. Редактирование WebApp не должно скрыто подтверждать родительскую продажу; исправленная позиция может стать `processed` и войти в выручку, пока неполные соседние позиции остаются на проверке.

Внутренние enum не показываются пользователю.
