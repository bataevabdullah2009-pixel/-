# AI Rules

LLM извлекает только факты из STT-текста. Количество и цена подтверждаются deterministic evidence layer. Нельзя выдумывать цену, количество или товар.

`confidence >= 0.80` требуется для автоматического `processed`. Низкая уверенность, отсутствующая цена/количество, странный текст или parser fallback дают `needs_review`.

Fallback parser должен исправлять LLM output, который вернул один длинный текст вместо массива товаров: валидные товары разделяются по evidence, а неполные фрагменты сохраняются отдельными review items.

`needs_review` не означает потерю продажи: bot сохраняет запись и показывает Telegram кнопки `✅ Подтвердить` и `❌ Отмена`. WebApp edit не должен скрыто подтверждать parent sale; исправленный item может стать `processed` и войти в выручку, пока неполные sibling items остаются в review.

Внутренние enum не показываются пользователю.
