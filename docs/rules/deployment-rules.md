# Правила развёртывания

- сначала применить миграции, затем развёртывать зависящий от них код;
- переменные задавать в Vercel/Supabase, не хранить значения в Git;
- после изменения environment выполнять redeploy;
- production Telegram использует webhook, локальная разработка — polling; одновременно их не запускать;
- webhook менять только по подтверждённой причине и затем проверять `getWebhookInfo`;
- команда установки webhook очищает pending updates, поэтому требует осознанного запуска;
- проверять отчёт и Mini App только на демонстрационных данных до production-hardening;
- иметь документированный rollback приложения и совместимый план миграции;
- не называть deployment проверенным по результату одной локальной сборки.
